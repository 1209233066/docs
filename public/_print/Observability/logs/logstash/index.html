logstash_install

使用filebeat收集linux操作系统日志
By wangendao | Friday, September 12, 2025

文档环境

os 版本： CentOS Linux release 7.9.2009 (Core)

filebeat版本：7.17.19

logstash版本：7.15.0

elasticsearch版本： 7.15.0

Logstash 使用jruby 语言开发，软件运行依赖jdk兼容矩阵，如果无法确认该使用什么版本可以下载带有jdk环境的logstash安装包。

安装java（非必须）

wget https://download.java.net/java/ga/jdk11/openjdk-11_linux-x64_bin.tar.gz
tar xf openjdk-11_linux-x64_bin.tar.gz -C /opt
ln -svf  /opt/{jdk-11,jdk}

cat>>/etc/profile<<'EOF'
export JAVA_HOME=/opt/jdk
export JAVA_JRE=$JAVA_HOME/jre
export CLASSPATH=$JAVA_HOME/lib:$JAVA_HOME/jre/lib
export PATH=$JAVA_HOME/bin:$JAVA_JRE/bin:$PATH:.
EOF

source /etc/profile
java -version

安装logstash

version=7.15.0
wget https://mirrors.huaweicloud.com/logstash/${version}/logstash-${version}-linux-x86_64.tar.gz

tar xf logstash-${version}-linux-x86_64.tar.gz -C /data/elk
ln  -svf /data/elk/logstash-${version}  /data/elk/logstash

测试运行

/data/elk/logstash/bin/logstash -e 'input { stdin {} } output { stdout { codec=>rubydebug }}'

cat >/data/elk/logstash/config/logstash.conf<<'EOF'
# Sample Logstash configuration for creating a simple
# Beats -> Logstash -> Elasticsearch pipeline.

input {
  beats {
    port => 5044
  }
}

output {
  elasticsearch {
    hosts => ["http://192.168.0.114:9200"]
    index => "%{[@metadata][beat]}-%{[@metadata][version]}-%{+YYYY.MM.dd}"
    #user => "elastic"
    #password => "changeme"
  }
}
EOF

tee /etc/systemd/system/logstash.service <<'EOF'
[Unit]
Description= logstash serveice
After=network.target
 
[Service]
WorkingDirectory=/data/elk/logstash

ExecStart=/data/elk/logstash/bin/logstash \
          -f /data/elk/logstash/config/logstash.conf \
          --config.reload.automatic

User=root
LimitNOFILE=65535
[Install]
WantedBy=multi-user.target
EOF

systemctl daemon-reload 
systemctl enable logstash --now
systemctl status logstash

logstash-input

logstash之input配置
By wangendao | Friday, September 12, 2025

文档环境

os 版本： CentOS Linux release 7.9.2009 (Core)

filebeat版本：7.17.19

logstash版本：7.15.0

elasticsearch版本： 7.15.0

INPUT 配置

redis

  • data_type[必选参数，数据类型为字符串] 可选值 list channel pattern_channel

  • key[必选参数，数据类型为字符串] 日志记录在那个名字的list /channel中

  • host[必选参数] redis绑定的服务ip

  • port redis绑定的服务端口，默认6379

  • password redis服务密码

  • threads [数据类型为整型]多线程消费redis数据，默认1

  • tags[所有类型intput都支持，数据类型为列表] 添加tag

    tags => ["linux","dev"]

  • add_field[所有类型intput都支持，数据类型为hash]添加自定义字段

    add_field => { 
    "env" => "pro" 
    "app" => "mobile"
    }

  • 示例

    cat >/data/elk/logstash/config/logstash.conf<<'EOF'
input {
  redis {
    data_type => "list"
    key => "logstash:beat:logstash"
    host => "192.168.0.161"
    password => "mypass"
    threads => 4
    add_field => { 
      "env" => "pro" 
      "app" => "mobile"
    }
    tags => ["linux","dev"]
  }
}

output {
  elasticsearch {
    hosts => ["http://192.168.0.114:9200"]
    index => "logstash-%{+YYYY.MM.dd}"
    #user => "elastic"
    #password => "changeme"
  }
}
EOF

file

将日志读取位置记录在sincedb 文件中，当重启logstash 后仍然可以继续读取。示例中记录在

/data/elk/logstash/data/plugins/inputs/file/.sincedb_452905a167cf4509fd08acb964fdb20c

  • path[必选参数，数据类型为数组]

  • start_position[数据类型字符串] logstash从哪里开始读取日志，可选参数begining end 默认为end。如果需要从头读取数据请设置为 begining

    该参数只作用在logstash 首次启动，也就是不存在sincedb 文件时

  • exclude[数据类型为数组] 排除匹配的文件不跟踪

    input {
    file {
        path => ["/var/log/*"]
        exclude => ["*.gz"]
        }
    }

  • tags[所有类型intput都支持，数据类型为列表] 添加tag

    tags => ["linux","dev"]

  • add_field[所有类型intput都支持，数据类型为hash]添加自定义字段

    add_field => { 
    "env" => "pro" 
    "app" => "mobile"
    }

  • 示例

    cat >/data/elk/logstash/config/logstash.conf<<'EOF'
input {
    file {
        path => ["/var/log/messages"]
		start_position => "beginning"
        }
    }
output {
  elasticsearch {
    hosts => ["http://192.168.0.114:9200"]
    index => "logstash-%{+YYYY.MM.dd}"
    #user => "elastic"
    #password => "changeme"
  }
}
EOF

exec 周期调用系统命令，logstash收集命令输出

  • command[必选参数，数据类型为字符串] 要执行的命令

  • interval 执行命令的间隔，单位s

    input {
  exec {
    command => "TERM=xterm /usr/bin/top -n 1 -b" 
    interval => 30
  }
}

  • schedule按照crontab格式周期执行命令

    input {
  exec {
    command => "TERM=xterm /usr/bin/top -n 1 -b" 
    schedule => "*/1 * * * *"
  }
}

  • tags[所有类型intput都支持，数据类型为列表] 添加tag

    tags => ["linux","dev"]

  • add_field[所有类型intput都支持，数据类型为hash]添加自定义字段

    add_field => { 
    "env" => "pro" 
    "app" => "mobile"
    }

  • 示例

    cat >/data/elk/logstash/config/logstash.conf<<'EOF'
input {
  exec {
    command => "TERM=xterm /usr/bin/top -n 1 -b" 
    interval => 30
  }
}
output {
  elasticsearch {
    hosts => ["http://192.168.0.114:9200"]
    index => "logstash-%{+YYYY.MM.dd}"
    #user => "elastic"
    #password => "changeme"
  }
}
EOF

http 用户 可以传递纯文本、JSON 或任何格式化数据

  • host 默认0.0.0.0

  • port 默认8080

  • threads 线程数默认 1

  • user/password 开启用户名密码认证

    cat >/data/elk/logstash/config/logstash.conf<<'EOF'
input {
  http {
	host => "0.0.0.0"
	port => "8080"
	threads => 4
    user => "log"
	password => "Q!123"
  }
}
output {
  elasticsearch {
    hosts => ["http://192.168.0.114:9200"]
    index => "logstash-%{+YYYY.MM.dd}"
    #user => "elastic"
    #password => "changeme"
  }
}
EOF

    curl -u 'log:Q!123'  -X POST --data '{"a":"123"}' 192.168.0.115:8080

kafka 从kafka消费信息

  • topics[数据类型数组] 默认值 ["logstash"]

  • topics_pattern[数据类型字符串] 正则模式匹配

  • consumer_threads 默认值 1

  • 示例

    cat >/data/elk/logstash/config/logstash.conf<<'EOF'
input {
  kafka {
    bootstrap_servers => "192.168.0.161:9092"   # Kafka 集群地址，多个用逗号分隔
    topics            => ["app-log"]            # 消费的 topic
    group_id          => "logstash-consumer"    # 消费者组 ID
    auto_offset_reset => "earliest"             # 从头开始（第一次无 offset 时）
    codec             => "json"                 # 消息体是 JSON 就加，纯文本可删
    decorate_events   => true                   # 给事件加 @metadata[kafka] 字段（topic/partition/offset）
  }
}
output {
  elasticsearch {
    hosts => ["http://192.168.0.114:9200"]
    index => "logstash-%{+YYYY.MM.dd}"
    #user => "elastic"
    #password => "changeme"
  }
}
EOF

logstash-output

logstash之output配置
By wangendao | Friday, September 12, 2025

文档环境

os 版本： CentOS Linux release 7.9.2009 (Core)

filebeat版本：7.17.19

logstash版本：7.15.0

elasticsearch版本： 7.15.0

OUTPUT配置

elasticsearch

  • hosts[非必须，数据类型数组] 默认值http://127.0.0.1:9200

  • index[非必须] 默认值 logstash-%{+YYYY.MM.dd}

  • user

  • password

  • 示例：

    cat >/data/elk/logstash/config/logstash.conf<<'EOF'
input {
  kafka {
    bootstrap_servers => "192.168.0.161:9092"   # Kafka 集群地址，多个用逗号分隔
    topics            => ["app-log","sys-log"]  # 消费的 topic
    group_id          => "logstash-consumer"    # 消费者组 ID
    auto_offset_reset => "earliest"             # 从头开始（第一次无 offset 时）
    codec             => "json"                 # 消息体是 JSON 就加，纯文本可删
    decorate_events   => true                   # 给事件加 @metadata[kafka] 字段（topic/partition/offset）
    consumer_threads => 5						# 开启多线程消费kafka
    # --------- SASL/SCRAM 认证（与 Kafka 保持一致） ---------
    #security_protocol => "SASL_PLAINTEXT"
    #sasl_mechanism    => "SCRAM-SHA-256"
    #sasl_jaas_config  => 'org.apache.kafka.common.security.scram.ScramLoginModule required
    #                      username="test"
    #                      password="pytc@2026";'

  }
}
output {
  elasticsearch {
    hosts => ["http://192.168.0.114:9200"]
    index => "logstash-%{+YYYY.MM.dd}"
    #user => "elastic"
    #password => "changeme"
  }
}
EOF

    上面的示例中所有topic日志都收集到了同一个索引中，接下来我们将不同日志分流到不同的index

    都有哪些@metadata 元数据

    cat >/data/elk/logstash/config/logstash.conf<<'EOF'
input {
  kafka {
    bootstrap_servers => "192.168.0.161:9092"   # Kafka 集群地址，多个用逗号分隔
    topics            => ["app-log","sys-log"]  # 消费的 topic
    group_id          => "logstash-consumer"    # 消费者组 ID
    auto_offset_reset => "earliest"             # 从头开始（第一次无 offset 时）
    codec             => "json"                 # 消息体是 JSON 就加，纯文本可删
    decorate_events   => true                   # 给事件加 @metadata[kafka] 字段（topic/partition/offset）
    consumer_threads => 5						# 开启多线程消费kafka
    # --------- SASL/SCRAM 认证（与 Kafka 保持一致） ---------
    #security_protocol => "SASL_PLAINTEXT"
    #sasl_mechanism    => "SCRAM-SHA-256"
    #sasl_jaas_config  => 'org.apache.kafka.common.security.scram.ScramLoginModule required
    #                      username="test"
    #                      password="pytc@2026";'
  }
}
output {
  if [@metadata][kafka][topic] == "app-log" {
    elasticsearch {
        hosts => ["http://192.168.0.114:9200"]
        index => "app-log-%{+YYYY.MM.dd}"
    }
  }
  if [@metadata][kafka][topic] == "sys-log" {
    elasticsearch {
        hosts => ["http://192.168.0.114:9200"]
        index => "sys-log-%{+YYYY.MM.dd}"
    }
  }
}
EOF

logstash-filter

logstash之filter配置
By wangendao | Friday, October 10, 2025

文档环境

os 版本： CentOS Linux release 7.9.2009 (Core)

filebeat版本：7.17.19

logstash版本：7.15.0

elasticsearch版本： 7.15.0

filter配置

grok 将非结构化的日志格式化为结构化可查询的内容

apache 访问日志示例

39.156.66.10 - - [27/Jun/2023:20:04:38 +0800] "GET /images/apache_pb.gif HTTP/1.1" 200 2326 "http://10.4.7.250/" "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/113.0.0.0 Safari/537.36 Edg/113.0.1774.57"

使用grok 对message执行解析

cat >/data/elk/logstash/config/logstash.conf<<'EOF'
input {
  kafka {
    bootstrap_servers => "192.168.0.161:9092"   # Kafka 集群地址，多个用逗号分隔
    topics            => ["app-log","sys-log"]  # 消费的 topic
    group_id          => "logstash-consumer"    # 消费者组 ID
    auto_offset_reset => "earliest"             # 从头开始（第一次无 offset 时）
    codec             => "json"                 # 消息体是 JSON 就加，纯文本可删
    decorate_events   => true                   # 给事件加 @metadata[kafka] 字段（topic/partition/offset）
  }
}

filter {
  grok {
    match => {"message" => "%{COMBINEDAPACHELOG}" }
  }
  geoip {
        source => "clientip"
  }
}

output {
  if [@metadata][kafka][topic] == "app-log" {
    elasticsearch {
        hosts => ["http://192.168.0.114:9200"]
        index => "app-log-%{+YYYY.MM.dd}"
    }
  }
  if [@metadata][kafka][topic] == "sys-log" {
    elasticsearch {
        hosts => ["http://192.168.0.114:9200"]
        index => "sys-log-%{+YYYY.MM.dd}"
    }
  }
}
EOF
